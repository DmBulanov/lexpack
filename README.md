# LexPack

Расширение Manifest V3 для поиска и массового сохранения документов из браузерной версии КонсультантПлюс. Основной сценарий — авторизованная система `online.consultant.ru`.

## Варианты

Проект содержит два независимо версионируемых варианта из одного исходного кода. Chrome и Chromium-Gost выпускаются готовыми ZIP.

| Вариант | Версия | Среда | Нативные загрузки |
|---|---:|---|---|
| Chrome | `0.9.1-chrome` | Google Chrome 120+ | Обычная скорость, одна попытка |
| Chromium-Gost | `0.9.1-gost` | Chromium-Gost 143+ | Стабилизация, пауза между актами, поздняя проверка и одна повторная попытка |

Название браузера отображается в карточке расширения и popup. В одном профиле одного браузера должна быть включена только одна копия LexPack.

## Возможности

- поиск по `online.consultant.ru` и переход к полной сгруппированной выдаче;
- выбор одной или нескольких судебных инстанций и объединение результатов без дублей;
- сбор уже открытой пользователем судебной категории без нового запроса;
- preview и произвольный выбор документов из подборки до 200 позиций;
- локальные профили с шаблонами имён и вложенных относительных папок;
- консервативное извлечение даты, номера дела, суда и типа акта для шаблонов;
- детерминированное разрешение внутренних коллизий до запуска;
- DOCX через нативное сохранение с последующей локальной очисткой данных КонсультантПлюс;
- PDF и RTF через нативное сохранение КонсультантПлюс без изменения содержимого;
- TXT и безопасный статический HTML из доступного содержимого страницы;
- сохраняемая очередь Manifest V3, остановка и восстановление;
- расширенный отчёт schema v2 с planned/actual результатом;
- локальная история `off` / `safe` / `detailed` и ручной retry через новый preview;
- обезличенная техническая диагностика.

На публичном `www.consultant.ru` доступны только область «Всё по запросу» и форматы TXT/HTML. Локальная программа `cons.exe` и Firefox не поддерживаются.

## Установка готовой сборки

После `npm run package` создаются:

- `dist/lexpack-chrome-0.9.1.zip`;
- `dist/lexpack-chromium-gost-0.9.1.zip`;
- `dist/release-index.json`;
- `dist/SHA256SUMS`.

Для Chrome и Chromium-Gost передавайте только ZIP нужного браузера. Архив необходимо распаковать в постоянную папку, затем выбрать эту папку через `chrome://extensions` → «Режим разработчика» → «Загрузить распакованное расширение». В каждом ZIP находится соответствующий `INSTALL.txt`.

## Разработка

`extension/` содержит общий исходный код и не загружается в браузер напрямую: в нём намеренно нет готового `manifest.json`. Сборщик объединяет общий manifest, конфигурацию варианта и runtime-файлы.

```bash
npm ci
npx playwright install chromium

npm run build             # оба Web Extension payload в build/
npm run build:chrome      # только build/chrome
npm run build:gost        # только build/chromium-gost
npm run check             # синтаксис, два manifest, unit и browser tests
npm run package           # оба ZIP, индекс релиза и SHA-256
npm run package:chrome
npm run package:gost
```

Для локальной проверки загружайте `build/chrome` либо `build/chromium-gost`. Каталоги `build/` и `dist/` генерируются автоматически и не редактируются вручную.

## Структура

```text
extension/                    # общий runtime и manifest.base.json
variants/
├── chrome/                   # версия, название, параметры и INSTALL для Chrome
└── chromium-gost/            # версия, название, параметры и INSTALL для Gost
scripts/                      # сборка, проверка и упаковка вариантов
test/                         # общие, вариантные и браузерные тесты
build/                        # генерируемые распакованные расширения
dist/                         # генерируемые контролируемые ZIP и SHA-256
```

Правила изменения, независимого версионирования, Git-тегов и отката описаны в [VARIANTS.md](VARIANTS.md).

## Данные и ограничения

LexPack работает локально и не имеет собственного сервера или телеметрии. Настройки, профили и выбранный режим истории сохраняются в профиле браузера, а активная подборка и очередь — во временном session storage. `safe`-история по умолчанию не хранит запросы, заголовки, URL, номера дел или список позиций; `detailed` включается только после предупреждения. Итоговый JSON-отчёт содержит названия документов, статусы и очищенные адреса источников; запрос включается только при `detailed`. Обращайтесь с отчётом как с частью рабочей выгрузки. Подробности находятся в [PRIVACY.md](PRIVACY.md).

Перед сохранением DOCX LexPack локально удаляет из OOXML-пакета логотипы, ссылки, служебные строки и свойства КонсультантПлюс в колонтитулах и метаданных. Текст основной части `word/document.xml` не изменяется; в нижнем колонтитуле остаётся динамический счётчик «Страница N из M». Если безопасно разобрать или очистить Word-файл не удалось, исходный DOCX не сохраняется. PDF и RTF по-прежнему загружаются без изменения содержимого.

Нативный экспорт зависит от интерфейса и механизма загрузки браузера. После обновлений КонсультантПлюс, Chrome или Chromium-Gost может потребоваться повторная калибровка. Перед выпуском обязательно проходите [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md).
