# LexPack

Расширение Manifest V3 для поиска и массового сохранения документов из браузерной версии КонсультантПлюс. Основной сценарий — авторизованная система `online.consultant.ru`.

## Варианты

Проект выпускает две независимо версионируемые сборки из одного исходного кода.

| Вариант | Версия | Среда | Нативные загрузки |
|---|---:|---|---|
| Chrome | `0.8.4-chrome` | Google Chrome 120+ | Обычная скорость, одна попытка |
| Chromium-Gost | `0.8.4-gost` | Chromium-Gost 143+ | Стабилизация, пауза между актами, поздняя проверка и одна повторная попытка |

Название браузера отображается в карточке расширения и popup. Одновременно в одном профиле должна быть включена только одна сборка LexPack: две активные копии конфликтуют при назначении имени загружаемого файла.

## Возможности

- поиск по `online.consultant.ru` и переход к полной сгруппированной выдаче;
- выбор одной или нескольких судебных инстанций и объединение результатов без дублей;
- сбор уже открытой пользователем судебной категории без нового запроса;
- экспорт до 200 документов за одну задачу;
- DOCX, PDF и RTF через нативное сохранение КонсультантПлюс;
- TXT и безопасный статический HTML из доступного содержимого страницы;
- сохраняемая очередь Manifest V3, остановка и восстановление;
- итоговый `LexPack-report-…json` и обезличенная диагностика.

На публичном `www.consultant.ru` доступны только область «Всё по запросу» и форматы TXT/HTML. Локальная программа `cons.exe`, Firefox и Safari не поддерживаются.

## Установка готовой сборки

После `npm run package` создаются:

- `dist/lexpack-chrome-0.8.4.zip`;
- `dist/lexpack-chromium-gost-0.8.4.zip`;
- `dist/release-index.json`;
- `dist/SHA256SUMS`.

Передавайте пользователю только ZIP нужного браузера. Архив необходимо распаковать в постоянную папку, затем выбрать эту папку через `chrome://extensions` → «Режим разработчика» → «Загрузить распакованное расширение». В каждом ZIP находится соответствующий `INSTALL.txt`.

## Разработка

`extension/` содержит общий исходный код и не загружается в браузер напрямую: в нём намеренно нет готового `manifest.json`. Сборщик объединяет общий manifest, конфигурацию варианта и runtime-файлы.

```bash
npm ci
npx playwright install chromium

npm run build             # обе распакованные сборки в build/
npm run build:chrome      # только build/chrome
npm run build:gost        # только build/chromium-gost
npm run check             # синтаксис, обе сборки, unit и browser tests
npm run package           # оба ZIP, индекс релиза и SHA-256
npm run package:chrome
npm run package:gost
```

Для локальной проверки загружайте `build/chrome` либо `build/chromium-gost`. Каталоги `build/` и `dist/` генерируются автоматически и не редактируются вручную.

## Структура

```text
extension/                    # общий runtime и manifest.base.json
variants/
├── chrome/                   # версия, название, параметры и INSTALL для Chrome
└── chromium-gost/            # версия, название, параметры и INSTALL для Gost
scripts/                      # сборка, проверка и упаковка вариантов
test/                         # общие, вариантные и браузерные тесты
build/                        # генерируемые распакованные расширения
dist/                         # генерируемые контролируемые ZIP и SHA-256
```

Правила изменения, независимого версионирования, Git-тегов и отката описаны в [VARIANTS.md](VARIANTS.md).

## Данные и ограничения

LexPack работает локально и не имеет собственного сервера или телеметрии. Настройки сохраняются в профиле браузера, а активная подборка и очередь — во временном session storage. Итоговый JSON-отчёт содержит запрос, названия документов, статусы и очищенные адреса источников, поэтому с ним следует обращаться как с частью рабочей выгрузки. Подробности находятся в [PRIVACY.md](PRIVACY.md).

Нативный экспорт зависит от интерфейса и событий загрузки браузера. После обновлений КонсультантПлюс, Chrome или Chromium-Gost может потребоваться повторная калибровка. Перед выпуском обязательно проходите [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md).
