# LexPack

Расширение Manifest V3 для поиска и массового сохранения документов из браузерной версии КонсультантПлюс. Основной сценарий — авторизованная система `online.consultant.ru`.

**Инструкция для пользователей:** [INSTRUCTION.md](INSTRUCTION.md)

**Конфиденциальность:** [PRIVACY.md](PRIVACY.md)

**Сообщить об уязвимости:** [SECURITY.md](SECURITY.md)

**Перед выпуском:** [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md)

Текущая версия — **LexPack 0.8.3**. Единственный установочный пакет для
пользователей — `dist/lexpack.zip`.

## Возможности

- при необходимости раскрыть компактную форму нового поиска прямо в popup;
- перейти из быстрого поиска в сгруппированную страницу **«Все результаты поиска»**;
- выбрать одну или несколько судебных инстанций и объединить их результаты без дублей;
- без нового запроса скачать открытый пользователем уровень судебной инстанции: действие показывается сверху popup, включая виртуализированные части списка;
- сохранить документы в DOCX, PDF, RTF, TXT или HTML в зависимости от страницы и возможностей адаптера;
- массово обработать уже найденный список;
- скачать открытый документ;
- автоматически выбрать все найденные документы (не более 200 за задачу), при необходимости уменьшить количество и получить итоговый JSON-отчёт;
- скопировать обезличенную диагностику для разбора ошибок интерфейса.

## Важные ограничения

- **DOCX / PDF / RTF** — нативный экспорт интерфейса авторизованного `online.consultant.ru`. Успех фиксируется только после события Chrome `complete`; `interrupted` считается ошибкой, а редкая неоднозначная загрузка выводится как **«требует проверки»**, потому что файл мог сохраниться.
- Поле **«Подпапка для документов и отчёта»** задаёт подпапку внутри основной папки загрузок Chrome для TXT / HTML, сопоставленных нативных загрузок и итогового JSON-отчёта. Основную папку и запрос места задаёт Chrome; JSON является обычным текстовым файлом и открывается в Блокноте или другом текстовом редакторе.
- На публичном `www.consultant.ru` доступны область **«Всё по запросу»** и форматы **TXT / HTML**. Фильтр «Судебная практика» и нативные DOCX / PDF / RTF публичным адаптером не поддерживаются.
- TXT / HTML строятся из доступного на странице содержимого и могут не сохранить сложную верстку, вложения или скрытые части документа.
- Состояние массовой очереди сохраняется между запусками Manifest V3 service worker. После восстановления всё равно сверяйте итоговый счётчик, позиции, требующие проверки, и ошибки с фактическими загрузками.
- Адаптер `online-app` зависит от CSS-селекторов сайта. После обновления интерфейса может потребоваться повторная калибровка.
- Автоматический выбор инстанций опирается на названия категорий полной выдачи. Отсутствующая для конкретного запроса категория пропускается; если отсутствуют все выбранные категории, задача завершается явной ошибкой.
- Защитный максимум 200 применяется к объединению выбранных инстанций. До поиска количество не задаётся: после сбора popup автоматически подставляет число найденных документов, а пользователь при необходимости уменьшает его.
- Если документов больше 200, popup явно сообщает об ограничении; для самостоятельно открытой категории с известным счётчиком показывает найденное и доступное к скачиванию количество.
- В ручном режиме полная выдача принимает только поддерживаемые уровни судебных инстанций; для законодательства, комментариев и путеводителей LexPack просит выбрать уровень судебной инстанции слева.

## Поддерживаемые среды

| Среда | Статус | Примечание |
|---|---|---|
| Google Chrome 120+ | Поддерживается | Рекомендуется актуальная стабильная версия. Chrome 120 — технический минимум для 30-секундных alarms и возобновляемой очереди. |
| Microsoft Edge 120+ (Chromium) | Best effort | Отдельно в CI не тестируется; названия настроек загрузок могут отличаться. |
| Firefox / Safari | Не поддерживаются | Используются Chrome Extensions API и Manifest V3. |
| Локальная программа `cons.exe` | Не поддерживается | Нужна браузерная версия КонсультантПлюс. |

## Установка

1. Получите `lexpack.zip` и распакуйте его в постоянную папку `LexPack`.
2. Убедитесь, что `manifest.json` находится непосредственно в этой папке, а не во второй вложенной папке.
3. Chrome → `chrome://extensions` → включите **Режим разработчика**.
4. Нажмите **«Загрузить распакованное расширение»** и выберите папку `LexPack`.
5. Закрепите тёмно-синий значок LexPack с белой буквой **L** и оранжевым загнутым углом.
6. Войдите через `login.consultant.ru` и дождитесь открытия `online.consultant.ru` до запуска поиска.

Подробный ежедневный сценарий и разбор типовых ситуаций находятся в [INSTRUCTION.md](INSTRUCTION.md).

## Что передавать другим юристам

Для установки передавайте только [dist/lexpack.zip](dist/lexpack.zip). Получатель распаковывает ZIP в постоянную папку `LexPack` и в `chrome://extensions` выбирает именно её — папку, где в корне лежит `manifest.json`; краткая памятка `INSTALL.txt` уже включена в архив. Исходники проекта, `node_modules`, тесты, профиль Chrome, логины, cookies, скачанные акты и JSON-отчёты для установки не передаются.

Если передаётся результат правового поиска, отправляйте выбранные DOCX/PDF/RTF по правилам вашей организации. `LexPack-report-…json` — дополнительная контрольная ведомость для сверки состава, статусов и источников; её передают при необходимости.

## Разработка и проверки

Проект zero-build: файлы из `extension/` загружаются браузером напрямую. Для полного набора проверок нужны Node.js 20+, npm, Chromium для Playwright и системные команды `zip`/`unzip`.

Локальный каталог проекта называется `LexPack`; для разработки Chrome загружает
`LexPack/extension`. Пользовательская распакованная папка также называется
`LexPack`, но содержит только файлы из установочного архива. Подпапка загрузок по
умолчанию — `LexPack`, а отчёт называется `LexPack-report-…json`.

```bash
npm ci
npx playwright install chromium
npm run check:syntax    # node --check для JS/CJS
npm run check:manifest  # JSON, обязательные поля и ссылки на файлы
npm test                # модульные тесты и проверки в реальном Chromium
npm run check           # все проверки выше
npm run package         # dist/lexpack.zip + проверка архива
```

После изменения runtime-файлов нажмите **Обновить** на карточке расширения в `chrome://extensions`. GitHub Actions выполняет `npm run check`, собирает ZIP и прикладывает его как artifact к каждому запуску CI.

Перед передачей ZIP пользователям пройдите [релизный чек-лист](RELEASE_CHECKLIST.md): он включает проверку совпадения версий, состава архива, поиска и нативных загрузок в реальном браузере.

## Данные и диагностика

Постоянно в профиле браузера сохраняются только область поиска, формат, подпапка и выбранные инстанции. Запрос используется во время активной задачи и входит в скачанный JSON-отчёт. Пояснения интерфейса показываются по кнопкам **i**, а статусы и ошибки остаются видимыми. Найденная подборка и очередь временно сохраняются в сессии Chrome, чтобы закрытие popup не уничтожило результат. В свёрнутом разделе **«Помощь при ошибке»** кнопка **«Скопировать диагностику»** копирует только версию и ограниченный набор структурных признаков без URL, запроса и заголовка страницы. **«Сбросить настройки и историю»** возвращает область поиска, формат, подпапку и инстанции к исходным значениям, удаляет текущую подборку и историю задачи, но не затрагивает скачанные файлы, настройки Chrome или вход в систему. Подробности — в [PRIVACY.md](PRIVACY.md).
