# LexPack Safari

Safari-вариант — отдельное macOS-приложение с Safari Web Extension. Оно использует тот же JavaScript из `extension/`, что Chrome и Chromium-Gost, но операции с файлами выполняет native handler приложения, потому что Safari не предоставляет совместимый `chrome.downloads`.

## Что хранится в Git

- `extension/` — общий Web Extension;
- `variants/safari/config.json` — версия, название, тайминги и выбор backend `safari-native`;
- `safari/LexPack Safari/LexPack Safari.xcodeproj` — постоянный Xcode-проект;
- `safari/LexPack Safari/LexPack Safari Extension/SafariWebExtensionHandler.swift` — сохранение TXT/HTML/JSON и подтверждение DOCX/PDF/RTF;
- `safari/LexPack Safari/LexPack Safari/LexPack Safari.entitlements` и `LexPack Safari Extension.entitlements` — sandbox и право принимать доступ к выбранной пользователем папке.

`build/safari` и `build/safari-derived` генерируются заново и в Git не хранятся. Xcode-проект ссылается на `build/safari`, поэтому после изменения общего JavaScript достаточно повторить сборку; копировать файлы расширения в проект вручную не нужно.

## Сборка

```bash
npm ci
npm run build:safari-app
```

Команда сначала создаёт Safari manifest и runtime в `build/safari`, затем выполняет проверочную компиляцию Xcode без релизной подписи. Готовность к запуску проверяется отдельно из Xcode:

1. Откройте `safari/LexPack Safari/LexPack Safari.xcodeproj`.
2. Выберите схему **LexPack Safari** и устройство **My Mac**.
3. Debug-конфигурация уже использует **Sign to Run Locally** без Team. Если Xcode всё же попросит подпись, оставьте Team = None и выберите локальную подпись; для Release позже потребуется команда разработчика.
4. Нажмите **Run**. Приложение покажет состояние расширения и кнопку перехода в настройки Safari.
5. В Safari → **Настройки → Расширения** включите **LexPack Safari**.
6. Разрешите расширению доступ к `consultant.ru`.
7. Откройте Safari → **Настройки → Сайты → Загрузки** и установите для `online.consultant.ru` значение **Разрешить**. Общую настройку для остальных сайтов можно оставить **Спросить**.
8. В приложении **LexPack Safari** нажмите **Выбрать папку загрузок Safari** и выберите ту же папку, которая указана в Safari → **Настройки → Основные → Папка для загрузки файлов** (обычно это «Загрузки»). При первом выборе macOS может отдельно запросить доступ приложения к этой папке.

Если Safari не показывает локально собранное расширение, включите меню разработчика в дополнительных настройках Safari, а затем **Develop → Allow Unsigned Extensions**. Используйте этот режим только для собственной проверочной сборки; настройка может сбрасываться после перезапуска Safari.

## Как сохраняются файлы

- Системное окно выбора выдаёт приложению ограниченный sandbox-доступ только к выбранной папке. Доступ сохраняется как bookmark и передаётся расширению по штатному каналу Safari native messaging.
- TXT, HTML и JSON передаются приложению через `nativeMessaging` и атомарно записываются в `<папка загрузок Safari>/<подпапка>`.
- Перед DOCX/PDF/RTF приложение делает снимок верхнего уровня выбранной папки загрузок Safari.
- После команды КонсультантПлюс приложение ждёт один новый файл ожидаемого типа, проверяет стабильность размера и перемещает его в нужную подпапку с контролируемым именем.
- Если одновременно появляются несколько подходящих файлов, LexPack ничего не перемещает и сообщает о неоднозначности.
- После нажатия «Стоп» уже запущенную загрузку Safari невозможно гарантированно отменить; такой файл может остаться в корне «Загрузок» и требует ручной проверки.

## Ручной тест прототипа

1. Проверить разрешение загрузок для `online.consultant.ru`, поиск и сбор открытой категории без скачивания.
2. Сохранить по одному TXT и HTML; проверить содержимое и отсутствие активных скриптов в HTML.
3. Сохранить по одному DOCX, PDF и RTF; проверить подпапку и имена.
4. Скачать подряд не менее семи DOCX и сверить число файлов с JSON-отчётом.
5. Во время одного документа запустить постороннюю загрузку того же типа: LexPack должен сообщить о неоднозначности и не перемещать файлы.
6. Проверить «Стоп», повторный запуск Safari, повторное использование сохранённого доступа и повторный выбор папки.

До прохождения этого теста Safari-вариант считается прототипом и не заменяет рабочие Chrome/Gost-сборки.

## Рассылка коллегам

Папка `build/safari` и обычный ZIP Web Extension для установки не подходят. Для распространения нужно создать Archive в Xcode, подписать приложение сертификатом подходящего канала распространения и выполнить нотарификацию Apple. Только после установки и проверки этого артефакта создаётся тег `safari-v…` и он добавляется в GitHub Release.

Официальные справочные материалы Apple: [упаковка Safari Web Extension](https://developer.apple.com/documentation/safariservices/packaging-a-web-extension-for-safari), [обмен сообщениями с приложением](https://developer.apple.com/documentation/safariservices/messaging-between-the-app-and-javascript-in-a-safari-web-extension), [распространение](https://developer.apple.com/documentation/safariservices/distributing-your-safari-web-extension).
