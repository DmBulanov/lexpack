# Релизный чек-лист Cons Export

Этот список применяется перед передачей ZIP пользователям или публикацией новой версии. Он не заменяет проверку прав на выгрузку материалов КонсультантПлюс.

## 1. Подготовка

- [ ] Рабочая копия содержит именно изменения, которые планируется выпускать; локальные секреты, скачанные документы и `dist/` не попадут в коммит.
- [ ] Версия в `package.json` совпадает с `extension/manifest.json`. Если версия указана в пользовательской документации (сейчас — в `PRIVACY.md`), она также обновлена.
- [ ] Для изменённых runtime-файлов расширение будет перезагружено через **«Обновить»** в `chrome://extensions`.

## 2. Автоматические проверки и пакет

В чистой или воспроизводимой среде выполните:

```bash
npm ci
npm run check
npm run package
```

- [ ] `npm run check` завершился успешно: синтаксис, manifest, модульные и Chromium-проверки прошли.
- [ ] `npm run package` создал `dist/cons-export.zip`, а `unzip -tq` из сценария упаковки подтвердил целостность архива.
- [ ] Состав ZIP проверен: в корне есть `manifest.json` и только runtime-файлы расширения, нужные для загрузки распакованного расширения.
- [ ] В ZIP нет `node_modules`, `test`, `.git`, `dist`, `.DS_Store`, отчётов, профилей браузера, документов, логинов, cookies и иных локальных данных.

## 3. Ручная проверка в авторизованном online.consultant.ru

Проверяйте только на собственной учётной записи и на запросах без чувствительных данных.

- [ ] Откройте уже авторизованный `online.consultant.ru`, введите тестовый запрос, отметьте судебные инстанции и выполните **«Найти»** с областью **«Всё по запросу»**. После быстрого поиска должна открыться полная сгруппированная выдача; краткие «наиболее подходящие» результаты не должны попасть в итоговый список.
- [ ] Не меняя запрос, повторите **«Найти»** с областью **«Судебная практика»**. Убедитесь, что активна именно эта область и список обновился, а не остался от предыдущего поиска. Это проверяет переход, когда адрес поиска может не измениться.
- [ ] Убедитесь, что после краткой выдачи открылась новая вкладка **«Все результаты поиска»** (`req=query`), а запрос в заголовке полной выдачи в точности совпадает с введённым.
- [ ] Выберите одновременно **«Высшие суды»** и **«Арбитражные суды округов»**. В результате присутствуют обе группы, а один и тот же акт не повторяется дважды.
- [ ] Отдельно выберите **«Арбитражные суды первой инстанции»** и категорию с пометкой **«Искать»**. Дождитесь формирования выборки и проверьте, что старая категория не попала в результат.
- [ ] Установите лимит меньше суммарного числа актов в выбранных инстанциях. Итоговый список не превышает лимит, а интерфейс сообщает об ограничении.
- [ ] Самостоятельно откройте категорию в дереве полной выдачи, затем используйте **«Собрать открытую категорию»**. Категория не переключается, длинный список собирается до лимита, прокрутка возвращается в исходное положение, а popup показывает «N из M» и причину усечения.
- [ ] Повторите переключение обратно на **«Всё по запросу»** с тем же запросом и проверьте, что итог снова взят из полной выдачи и выбранных инстанций.
- [ ] Выполните заведомо пустой запрос или фильтр. Расширение показывает пустой результат и не предлагает экспортировать старые строки.
- [ ] Откройте один документ из выдачи и проверьте TXT или HTML через **«Сохранить текущий»**: созданный файл соответствует открытому документу.
- [ ] На отдельном одном документе проверьте DOCX, PDF или RTF, доступный по лицензии: Chrome получает одну ожидаемую нативную загрузку, она завершается `complete`, а файл оказывается в выбранной подпапке (если путь не был изменён в диалоге Chrome).
- [ ] Во время нативного экспорта запустите независимую загрузку из другой вкладки. Она не должна быть засчитана как файл Cons Export и не должна быть переименована расширением.

## 4. Публичная страница и ограничения адаптера

- [ ] На поддерживаемой странице `www.consultant.ru` выполните **«Найти»** с областью **«Всё по запросу»** и убедитесь, что выдача соответствует запросу.
- [ ] На публичной странице TXT/HTML доступен для текущего документа; **«Судебная практика»** и DOCX/PDF/RTF корректно обозначены как недоступные, без попытки экспорта.

## 5. Остановка и восстановление

- [ ] Запустите небольшую массовую задачу и нажмите **«Стоп»** во время обработки. Очередь прекращается, временная вкладка закрывается, а текущая нативная загрузка отменяется либо явно получает терминальный статус.
- [ ] Нажмите **«Стоп»** ещё раз и дождитесь нескольких секунд: повторный сигнал не запускает новую загрузку и не отменяет чужие загрузки.
- [ ] После перезапуска service worker или браузера проверьте сохранённую незавершённую задачу: возобновляется только ожидающая очередь, а итоговые счётчики и JSON-отчёт согласуются с `chrome://downloads`.

## 6. Диагностика и конфиденциальность

- [ ] Нажмите **«Разведка UI»** только на тестовой странице и проверьте результат перед копированием или передачей.
- [ ] Диагностика нативной загрузки и остановки, если она показана, содержит только закрытые коды (`NM_*` для сопоставления и `DG_*` для остановки), время и укрупнённый счётчик `0`/`1`/`many`.
- [ ] В диагностике нет запроса, названия документа, URL, referrer, имени файла, идентификаторов загрузки, cookies или текста ошибки браузера.
- [ ] Итоговый JSON-отчёт проверен отдельно: он предназначен для состава выгрузки и не должен служить каналом диагностических событий загрузок.

## 7. Передача результата

- [ ] Пользователю передаётся `dist/cons-export.zip` или согласованный коммит, а не профиль браузера и не папка со скачанными документами.
- [ ] В релизной заметке указаны версия, поддерживаемый минимум Chrome 120 и известные ограничения форматов/публичного адаптера.
- [ ] При обнаружении проблемы, связанной с данными, загрузками или обходом ограничений домена, используйте порядок из [SECURITY.md](SECURITY.md), не прикладывая реальные материалы или диагностический вывод с данными.
