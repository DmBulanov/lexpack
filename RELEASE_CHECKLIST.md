# Релизный чек-лист LexPack

## 1. Источник и версии

- [ ] Изменения находятся в общем `extension/` либо в нужном `variants/*`; `build/` и `dist/` вручную не редактировались.
- [ ] `variants/chrome/config.json` содержит актуальные версию, `versionName`, минимальный Chrome и имя Chrome-архива.
- [ ] `variants/chromium-gost/config.json` содержит актуальные версию, `versionName`, минимальный Chromium-Gost и имя Gost-архива.
- [ ] Изменение общего кода оценено для обоих вариантов; изменение только одного варианта не повысило второй без причины.
- [ ] В Git нет логинов, cookies, скачанных актов, рабочих JSON-отчётов или секретов.

## 2. Автоматические проверки

```bash
npm ci
npx playwright install chromium
npm run check
npm run package
```

- [ ] Unit-тесты прошли.
- [ ] Smoke-тест загрузил Chrome и Chromium-Gost и сверил их названия, версии, popup и service worker.
- [ ] Созданы оба версионированных ZIP.
- [ ] `dist/release-index.json` перечисляет только ожидаемые варианты и версии.
- [ ] SHA-256 каждого ZIP совпадает с `dist/SHA256SUMS`.
- [ ] `unzip -tq` подтвердил целостность архивов.
- [ ] В корне каждого ZIP находятся `manifest.json`, `BUILD-INFO.json` и соответствующий `INSTALL.txt`; исходного `manifest.base.json` там нет.

## 3. Общая ручная проверка

- [ ] В одном тестовом профиле включена только одна сборка LexPack.
- [ ] Popup открывается без ошибок, показывает правильные вариант и версию.
- [ ] Поиск открывает полную выдачу и собирает выбранные судебные инстанции без дублей.
- [ ] Уже открытая судебная категория собирается без нового поиска; несудебная категория не экспортируется.
- [ ] Подборка сохраняется после закрытия popup.
- [ ] «Настроить выгрузку» открывает planner, а выбор 2-й и 5-й строк даёт export indexes 01 и 02 при source indexes 2 и 5.
- [ ] Создание, дублирование, переименование, удаление и повторное открытие профиля работают; встроенный профиль удалить нельзя.
- [ ] Неизвестный токен блокирует запуск; отсутствующие метаданные видны как предупреждения; nested folder остаётся относительным.
- [ ] Две внутренние коллизии разрешаются в preview как имя и `имя (2)` одинаково во всех вариантах.
- [ ] TXT/HTML открытого документа сохраняются корректно.
- [ ] DOCX/PDF/RTF получают ожидаемые имена и подпапку.
- [ ] Посторонняя загрузка из другой вкладки не засчитывается и не переименовывается.
- [ ] «Стоп» прекращает очередь и не отменяет чужие загрузки.
- [ ] После перезапуска service worker очередь использует исходный snapshot профиля и те же planned paths.
- [ ] Итоговый JSON schema v2 согласуется с фактическими файлами, различает planned/actual и содержит только очищенные диагностические события без download ids.
- [ ] Safe history в persistent storage не содержит query/title/URL/case; detailed retry открывает preview и не стартует автоматически.
- [ ] Общий reset удаляет профили, подборку, очередь и историю, но не файлы на диске.

## 4. Проверка Chrome

- [ ] Загружен именно `build/chrome`; карточка показывает `Chrome` и ожидаемый `versionName`.
- [ ] Серия нативных загрузок идёт без Gost-паузы и без `NM_RETRY`.
- [ ] Результат проверен на заявленной минимальной либо поддерживаемой версии Google Chrome.

## 5. Проверка Chromium-Gost

- [ ] Загружен именно `build/chromium-gost`; карточка показывает `Chromium-Gost` и ожидаемый `versionName`.
- [ ] Серия не менее чем из семи DOCX сохраняет все акты последовательно.
- [ ] Между подтверждёнными загрузками выдерживается Gost-пауза.
- [ ] Пропущенный старт получает не более одной повторной попытки `NM_RETRY`.
- [ ] Запоздавшая первая загрузка не создаёт дубль.
- [ ] Результат проверен на рабочей версии Chromium-Gost для Windows.

## 6. Передача и контроль

- [ ] Пользователь получает только ZIP нужного браузера, а не весь проект.
- [ ] Передаваемые имя файла, версия и SHA-256 записаны в релизной заметке.
- [ ] Предыдущий ZIP сохранён неизменным для отката.
- [ ] После подтверждения релиза создан соответствующий Git-тег `chrome-v…` или `chromium-gost-v…`.
- [ ] Документы и отчёты передаются и хранятся по внутренним правилам организации.
