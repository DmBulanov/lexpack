# Варианты Chrome и Chromium-Gost

## Основной принцип

В репозитории хранится один общий Web Extension и две небольшие конфигурации. Полные копии расширения и долгоживущие Git-ветки для браузеров не используются: это предотвращает расхождение исправлений.

Источники истины:

- `extension/` — общее поведение, интерфейс, адаптеры и тестируемые контракты;
- `extension/manifest.base.json` — общие разрешения и точки входа Manifest V3;
- `variants/chrome/config.json` — версия и политика Chrome;
- `variants/chromium-gost/config.json` — версия и политика Chromium-Gost;
- `variants/*/INSTALL.txt` — памятка конкретного варианта.

`build/` и `dist/` — производные каталоги. Их можно удалить и полностью восстановить командами сборки; вручную они не редактируются и в Git не коммитятся.

## Матрица различий

| Параметр | Chrome | Chromium-Gost |
|---|---:|---:|
| Отображаемая версия | `0.9.2-chrome` | `0.9.2-gost` |
| Файловый API | `chrome.downloads` | `chrome.downloads` |
| Ожидание готовности команды | 0 мс | 1500 мс |
| Пауза между актами | 0 мс | 5000 мс |
| Поздняя проверка после таймаута | 0 мс | 2500 мс |
| Максимум попыток нативного старта | 1 | 2 |
| Окно сопоставления загрузки | 35000 мс | 40000 мс |

Все временные значения задаются в `nativeDownloads` соответствующего `config.json` и попадают в генерируемый `shared/variant-config.js`.

## Где редактировать

- Общая функция или исправление для обоих браузеров: редактировать `extension/`.
- Только тайминги, название или версия: редактировать нужный `variants/*/config.json`.
- Существенно различающееся поведение: сначала оформить отдельной стратегией в общем коде; не размножать общий JavaScript по папкам браузеров.
- Пользовательская памятка конкретного варианта: редактировать `variants/*/INSTALL.txt`.
- Не редактировать `build/*/manifest.json`, `shared/variant-config.js`, `BUILD-INFO.json` и ZIP: следующая сборка их перезапишет.

## Версионирование

Номера вариантов независимы. Числовой `manifest.version` совместим с Web Extension, а `manifest.versionName` добавляет канал:

- Chrome: `0.9.2` и `0.9.2-chrome`;
- Chromium-Gost: `0.9.2` и `0.9.2-gost`.

Если изменена только политика одного варианта, повышается только его версия. Если общий код меняет поведение обоих вариантов, их версии пересматриваются отдельно. Имена ZIP включают вариант и числовую версию.

Рекомендуемые Git-теги после успешной проверки реального браузера:

```text
chrome-v0.9.2
chromium-gost-v0.9.2
```

Тег должен указывать на коммит, из которого создан фактически переданный релиз.

## Сборка и проверка

```bash
npm run build
npm run check
npm run package
```

Контроль состоит из следующих уровней:

1. Обе конфигурации проверяются на обязательные поля, версии и согласованность временных окон.
2. Для каждого варианта создаётся отдельный `manifest.json`.
3. Unit-тесты подтверждают общие контракты и различия политик загрузки.
4. Smoke-тест загружает Chrome и Chromium-Gost в реальный Chromium.
5. `npm run package` проверяет целостность двух ZIP и создаёт `release-index.json` с SHA-256 и `SHA256SUMS`.

## Выпуск и откат

Перед передачей сравните имя ZIP, `versionName` в `release-index.json` и SHA-256. Предыдущие ZIP храните неизменными. Обновление распакованного варианта выполняется заменой его постоянной папки и кнопкой «Обновить» в `chrome://extensions`.

В одном профиле одного браузера нельзя одновременно включать две копии LexPack: они будут конкурировать за одну и ту же задачу и загрузку.
