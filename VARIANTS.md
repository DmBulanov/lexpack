# Варианты Chrome, Chromium-Gost и Safari

## Основной принцип

В репозитории хранится один общий Web Extension и три небольшие конфигурации. Полные копии расширения и долгоживущие Git-ветки для браузеров не используются: это предотвращает расхождение исправлений.

Источники истины:

- `extension/` — общее поведение, интерфейс, адаптеры и тестируемые контракты;
- `extension/manifest.base.json` — общие разрешения и точки входа Manifest V3;
- `variants/chrome/config.json` — версия и политика Chrome;
- `variants/chromium-gost/config.json` — версия и политика Chromium-Gost;
- `variants/safari/config.json` — версия и политика Safari;
- `safari/` — Xcode-приложение, native handler и сохраняемый доступ к выбранной папке загрузок Safari;
- `variants/*/INSTALL.txt` — памятка конкретного варианта.

`build/` и `dist/` — производные каталоги. Их можно удалить и полностью восстановить командами сборки; вручную они не редактируются и в Git не коммитятся. Xcode-проект в `safari/`, наоборот, является исходным кодом и хранится в Git.

## Матрица различий

| Параметр | Chrome | Chromium-Gost | Safari |
|---|---:|---:|---:|
| Отображаемая версия | `0.8.4-chrome` | `0.8.4-gost` | `0.8.4-safari` |
| Файловый backend | `chrome.downloads` | `chrome.downloads` | native handler приложения |
| Ожидание готовности команды | 0 мс | 1500 мс | 500 мс |
| Пауза между актами | 0 мс | 5000 мс | 1500 мс |
| Поздняя проверка после таймаута | 0 мс | 2500 мс | 0 мс |
| Максимум попыток нативного старта | 1 | 2 | 1 |
| Окно сопоставления загрузки | 35000 мс | 40000 мс | 45000 мс |

Все значения задаются в `nativeDownloads` соответствующего `config.json` и попадают в генерируемый `shared/variant-config.js`. Поле `fileBackend` выбирает Chromium API либо мост Safari.

## Где редактировать

- Общая функция или исправление для всех браузеров: редактировать `extension/`.
- Только тайминги, название или версия: редактировать нужный `variants/*/config.json`.
- Существенно различающееся поведение: сначала оформить отдельной стратегией в общем коде; не размножать общий JavaScript по папкам браузеров.
- Только доступ Safari к локальным файлам и сообщения от расширения к приложению: редактировать `safari/LexPack Safari/LexPack Safari Extension/SafariWebExtensionHandler.swift` и соответствующие entitlements.
- Пользовательская памятка конкретного варианта: редактировать `variants/*/INSTALL.txt`.
- Не редактировать `build/*/manifest.json`, `shared/variant-config.js`, `BUILD-INFO.json` и ZIP: следующая сборка их перезапишет.

## Версионирование

Номера вариантов независимы. Числовой `manifest.version` совместим с Web Extension, а `manifest.versionName` добавляет канал:

- Chrome: `0.8.4` и `0.8.4-chrome`;
- Chromium-Gost: `0.8.4` и `0.8.4-gost`;
- Safari: `0.8.4` и `0.8.4-safari`.

Если изменена только политика одного варианта, повышается только его версия. Если общий код меняет поведение нескольких вариантов, версии затронутых конфигураций пересматриваются отдельно. Имена ZIP включают вариант и числовую версию; Safari распространяется как подписанное приложение, а не как Chrome ZIP.

Рекомендуемые Git-теги после успешной проверки реального браузера:

```text
chrome-v0.8.4
chromium-gost-v0.8.4
safari-v0.8.4
```

Тег должен указывать на коммит, из которого создан фактически переданный релиз. Safari-тег ставится только после проверки подписанного и нотарифицированного приложения.

## Сборка и проверка

```bash
npm run build
npm run check
npm run package
npm run build:safari-app   # отдельно на macOS с Xcode
```

Контроль состоит из следующих уровней:

1. Все три конфигурации проверяются на обязательные поля, версии и согласованность временных окон.
2. Для каждого варианта создаётся отдельный `manifest.json`; Safari manifest не содержит `downloads` и `offscreen`, но содержит `nativeMessaging`.
3. Unit-тесты подтверждают различия файловых backend и политик загрузки.
4. Smoke-тест загружает Chrome и Chromium-Gost в реальный Chromium. Safari нельзя достоверно проверить в Chromium, поэтому он проходит отдельную сборку Xcode и ручной тест в Safari.
5. `npm run package` проверяет целостность только двух Chromium ZIP и создаёт `release-index.json` с SHA-256 и `SHA256SUMS`.
6. `npm run build:safari-app` пересобирает `build/safari` и компилирует приложение без релизной подписи. Для реальной установки его запускают из Xcode; для рассылки создают подписанный архив приложения.

## Выпуск и откат

Для Chrome/Gost перед передачей сравните имя ZIP, `versionName` в `release-index.json` и SHA-256. Предыдущие ZIP храните неизменными. Обновление распакованного варианта выполняется заменой его постоянной папки и кнопкой «Обновить» в `chrome://extensions`.

Для Safari исходником остаётся Git-коммит, а распространяемым артефактом — подписанное и нотарифицированное приложение конкретной версии. Предыдущий установочный образ или архив приложения также хранится неизменным для отката. Папка `build/safari` сама по себе не является устанавливаемым релизом.

В одном профиле одного браузера нельзя одновременно включать две копии LexPack: они будут конкурировать за одну и ту же задачу и загрузку.
